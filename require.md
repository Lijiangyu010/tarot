# Mystic Weave Tarot 静态网站需求文档（适配移动端）
## 一、项目概述
### 1.1 项目名称
Mystic Weave Tarot（神秘编织塔罗）
### 1.2 项目定位
**纯前端静态网站**，基于莱德韦特（Rider-Waite）塔罗体系，提供「三张牌阵（过去-现在-未来）」在线占卜服务，无后端/数据库依赖，适配桌面端与移动端，打造轻量化、沉浸式的塔罗占卜体验，可直接部署至 GitHub Pages 等静态托管平台。绑定仓库。https://github.com/Lijiangyu010/tarot/
### 1.3 核心目标
- 纯前端实现「洗牌→切牌→抽牌→翻牌→解读」的完整占卜流程，无任何后端接口或数据存储需求
- 界面风格契合莱德韦特塔罗的神秘紫色调性，布局清爽规整，同时适配桌面与移动端
- 保证动画流畅、资源加载快速，静态资源可直接部署访问

## 二、核心功能需求（静态可实现版）
### 2.1 导航与基础功能
| 功能模块 | 功能描述 |
|----------|----------|
| 顶部导航栏 | 桌面端：左侧 Logo + 中间导航项（HOME、READINGS）+ 右侧截图按钮；移动端：Logo + 汉堡菜单（点击展开导航项）+ 截图按钮，移除原 JOURNAL/SHOP/COMMUNITY（静态无法实现用户数据/商品/社区交互） |
| 截图按钮 | 截图塔罗牌网站页面，供他处解析 |
| 搜索/用户中心 | 完全移除（静态网站无后端搜索/用户数据存储能力） |

### 2.2 核心占卜功能（纯前端实现）
| 流程步骤 | 功能描述 |
|----------|----------|
| 洗牌 | 点击「洗牌」按钮，前端触发牌堆洗牌动画：牌堆扇形展开→收拢为牌堆，同时用 JS 随机打乱牌组数组顺序；洗牌期间按钮置灰不可重复点击 |可多次洗牌，直到点击去切牌，叠放好牌堆|
| 切牌 | 仅在洗牌完成后解锁「切牌」按钮，点击后触发切牌动画：牌堆分割为两叠→重新堆叠为一叠，JS 随机确定切牌位置；切牌期间按钮置灰 ||可多次切牌，直到点击占卜，叠放好牌堆|
| 开始占卜 | 仅在切牌完成后解锁「抽牌」按钮，点击后 JS 从随机牌组中抽取 1 张牌，触发抽牌动画：牌沿光轨依次飞入底部「1、2、3」放牌区 |一次抽一张|
| 翻牌 | 牌到达放牌区后，触发翻牌动画：从紫色太阳纹牌背 3D 翻转至莱德韦特牌面，伴随光晕缩放效果；翻牌完成后，前端根据抽到的牌显示**预定义的核心解读关键词**（如「The High Priestess - Intuition & Mystery」） |
| 问题与解读 | 桌面端：右侧侧边栏展示「Current Question」临时输入框（仅本地生效，刷新后丢失）+「Interpretation」解读区；移动端：改为底部折叠面板，点击展开/收起，随翻牌顺序实时更新解读内容 |

### 2.3 扩展功能（静态可实现）
| 功能模块 | 功能描述 |
|----------|----------|
| 塔罗牌义手册 | 静态页面，展示所有莱德韦特塔罗牌的牌面+基础牌义（纯文本+图片，无搜索/交互） |

## 三、界面设计需求（响应式适配）
### 3.1 整体风格
- **主色调**：深紫色背景 + 金色/米白色点缀，保留星空、星盘、星月符文等神秘学视觉元素
- **视觉规范**：莱德韦特经典牌面、紫色太阳纹牌背、复古雕花边框，风格统一且简洁无冗余
- **响应式原则**：桌面端优先，同时适配平板（768-1023px）、手机（<768px），保证小屏幕下内容可读、操作可触

### 3.2 布局结构
#### 桌面端（≥1024px）
1.  **顶部栏**：左侧 Logo（星月+太阳图标）→ 中间导航项（HOME/READINGS）→ 右侧分享按钮
2.  **主内容区（上半部分）**：
    - 标题：「THE THREE-CARD SPREAD READING」
    - 操作按钮区：「SHUFFLE DECK」（紫色发光）、「CUT DECK」（金色边框）、「START READING」（浅紫边框），按钮 hover 有光影反馈
    - 牌堆展示区：初始为紫色太阳纹牌堆，洗牌后展开为扇形莱德韦特牌面
3.  **主内容区（下半部分）**：
    - 标题：「DRAWING AREA」
    - 三张放牌区：等大水平分布，标注「1. PAST」「2. PRESENT」「3. FUTURE」，紫色雕花边框
4.  **右侧侧边栏**：「Current Question」临时输入框 +「Interpretation」解读区

#### 移动端（<768px）
1.  **顶部栏**：Logo + 汉堡菜单（点击展开导航）+ 分享按钮，导航项垂直排列
2.  **主内容区**：垂直流式布局：标题→操作按钮→牌堆→放牌区，按钮宽度自适应屏幕，大小≥44×44px（触摸友好）
3.  **放牌区**：三张牌位垂直堆叠或水平紧凑排列，保证小屏幕下完整展示
4.  **问题与解读区**：改为底部折叠面板，默认收起，点击展开查看/输入问题、查看解读，避免占用核心操作空间

### 3.3 视觉细节
- 牌背：紫色底+金色太阳纹，复古雕花边框
- 牌面：严格遵循莱德韦特塔罗经典插画，保证识别度
- 动画：洗牌扇形展开/收拢、切牌堆叠、抽牌光轨拖影、翻牌 3D 翻转+光晕，整体柔和流畅，无卡顿

## 四、交互需求（纯前端可实现）
### 4.1 流程控制
- 按钮状态由 JS 前端控制：未洗牌时「CUT DECK」「START READING」置灰；洗牌后解锁「CUT DECK」；切牌后解锁「START READING」
- 动画执行期间，对应按钮置灰，防止重复操作

### 4.2 移动端交互优化
- 汉堡菜单：点击展开/收起导航项，触摸区域≥44×44px
- 折叠面板：问题与解读区默认收起，点击标题展开，避免遮挡占卜操作
- 按钮间距：≥8px，防止误触
- 文字大小：≥14px，保证小屏幕可读性

### 4.3 牌面交互
- 洗牌动画：牌堆扇形展开→快速收拢，模拟线下洗牌手感
- 抽牌动画：牌从牌堆沿金色光轨缓慢飞入对应放牌区，带动态拖影
- 翻牌动画：牌背→牌面 3D 翻转，伴随缩放+光晕，突出神秘感

## 五、技术需求（纯前端静态）
### 5.1 技术栈
- 基础：HTML5 + CSS3 + Vanilla JS（无框架，便于静态部署）
- 动画：CSS3 transitions/animations + JS 控制动画时序
- 响应式：CSS Flexbox/Grid + 媒体查询，适配多端
- 无后端依赖：所有逻辑、数据、解读均前端预定义，无 API 调用/数据库操作

### 5.2 资源优化
- 图片：塔罗牌面转换为 WebP 格式，装饰元素（星盘/星座）使用 SVG 矢量图
- 懒加载：非首屏图片（如牌义手册）使用 `loading="lazy"`
- 体积控制：静态资源总大小≤2MB，保证移动端加载速度

### 5.3 兼容性
- 支持所有现代浏览器（Chrome/Firefox/Safari/Edge）
- 不兼容 IE 浏览器，页面提示用户升级浏览器

---

## 六、非功能需求
### 6.1 性能要求
- 首屏加载时间 ≤ 2s（静态资源轻量化）
- 动画帧率 ≥ 50fps，无卡顿、掉帧
- 移动端 3G/4G 网络下可正常加载

### 6.2 可用性要求
- 操作流程直观：洗牌→切牌→抽牌→解读，新手无需学习即可上手
- 移动端友好：按钮可触、文字可读、布局不拥挤，无横向滚动

### 6.3 可扩展性
- 可后续添加更多静态牌阵（纯前端实现）
- 可扩展静态牌义手册内容
- 可替换为其他塔罗体系（如透特塔罗）

---

## 七、验收标准
1.  **纯前端可运行**：无后端/数据库依赖，可直接部署至 GitHub Pages 等静态平台，打开即可使用
2.  **流程完整**：洗牌→切牌→抽牌→翻牌→解读流程可正常执行，按钮状态按流程解锁，无逻辑错误
3.  **多端适配**：桌面/平板/手机端布局自适应，无内容溢出、错位，操作流畅
4.  **视觉一致**：界面风格与参考图匹配，紫色主色调、莱德韦特牌面、动画效果符合预期
5.  **性能达标**：首屏加载≤2s，动画流畅，移动端无加载卡顿
